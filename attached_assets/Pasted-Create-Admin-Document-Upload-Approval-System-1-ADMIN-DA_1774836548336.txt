Create Admin Document Upload + Approval System

-----------------------------------
1. ADMIN DASHBOARD - DOCUMENT MANAGEMENT
-----------------------------------

Create a new section in admin panel:

"Customer Documents"

Features:

- Search user (name / phone / bookingId)
- Open customer profile
- View all documents:

  • Passport
  • Aadhar
  • PAN
  • Health Certificate
  • Visa
  • Ticket

-----------------------------------
2. ADMIN UPLOAD DOCUMENTS
-----------------------------------

Allow admin to upload:

- Visa (PDF/Image)
- Flight Ticket (PDF/Image)

UI:

- Upload button for each document
- Show preview
- Replace existing file if re-uploaded

Store in DB:
- visaDocument (URL)
- ticketDocument (URL)

-----------------------------------
3. APPROVAL SYSTEM
-----------------------------------

Add approval status fields:

For each document:

- passportStatus (pending / approved / rejected)
- aadharStatus
- panStatus
- healthStatus
- visaStatus
- ticketStatus

Admin actions:

- Approve ✅
- Reject ❌
- Add comment (reason)

-----------------------------------
4. DATABASE UPDATE
-----------------------------------

Update customer_profiles table:

Add:

- visaDocument
- ticketDocument

- passportStatus
- aadharStatus
- panStatus
- healthStatus
- visaStatus
- ticketStatus

- adminComment (optional)

-----------------------------------
5. CUSTOMER APP UPDATE
-----------------------------------

In "My Documents" screen:

For each document show:

- Status:
   🟡 Pending
   🟢 Approved
   🔴 Rejected

- If rejected → show reason

-----------------------------------
6. AUTO NOTIFICATIONS (VERY IMPORTANT)
-----------------------------------

Trigger notification when:

- Document Approved
- Document Rejected
- Visa Uploaded
- Ticket Uploaded

Example:

Title:
"Visa Approved"

Body:
"Assalamu Alaikum {{name}}, your visa has been approved. Please check your documents."

-----------------------------------
7. API ENDPOINTS
-----------------------------------

Admin APIs:

POST /api/admin/upload-document
POST /api/admin/update-status

User APIs:

GET /api/user/documents

-----------------------------------
8. SECURITY
-----------------------------------

- Only admin can upload/approve
- User can only view own documents

-----------------------------------
9. UI REQUIREMENTS
-----------------------------------

- Clean table layout in admin
- Status color badges
- Upload buttons per document
- Mobile responsive

-----------------------------------

Ensure:
- No duplicate uploads
- Status updates reflect instantly
- Notifications sent only once
